Overview

Built to help users understand device health, setup, and behavior quickly.

The app combines read-only device insights with guided tests, so users can check hardware details, software state, app inventory, and common diagnostics from one place.

Hardware and system

Model, manufacturer, board, build details, Android version, patch level, kernel, memory, storage, display, battery, network, and sensor information.

Installed apps and package insights

Browse installed apps with filtering, search, version details, install dates, source type, and APK size formatting for user and supported system packages.

Interactive diagnostics

Run checks for display, multi-touch, speaker, microphone, flashlight, vibration, charging, network, and supported sensors with visible pass or fail state.

Features

Simple, readable, and focused on real device information.

Dashboard summaries

Quick-glance cards for device profile, battery, RAM, storage, display, sensors, installed apps, and diagnostics progress.

System-driven theming

The app follows Android light and dark mode automatically, and this site mirrors the same palette for a consistent brand feel.

Diagnostics memory

Test status is persisted so users can return to the app and still see which checks have passed, failed, or are still pending.

Widgets

Home-screen widgets surface RAM usage and a compact device summary without opening the full app.

Firebase support

Firebase Analytics, Crashlytics, and Remote Config help measure stability, understand product usage, and manage controlled improvements.

Privacy-first transparency

The app website clearly explains what is read on-device, what is sent to Firebase, and why each declared permission exists.

Privacy

Clear permission and data-use language for Google Play review.

The privacy policy now covers Firebase Analytics, Crashlytics, Remote Config, local diagnostics state, installed app visibility, and each manifest permission with a direct user-facing justification.

Most device information is read and displayed on the device itself.

Firebase services are used for analytics, crash reporting, and configuration delivery.

Camera, microphone, vibration, and package visibility are tied to explicit diagnostic or inventory features.
