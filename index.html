Overview

Built to explain device health, software state, and app behavior without the clutter.

The app combines read-only device insights with export tools, guided diagnostics, and live sensor feedback so users can understand both the phone and the apps running on it.

Five focused tabs

Dashboard, Hardware, System, Apps, and Tests keep the experience organized while still surfacing live battery, RAM, CPU, network, and diagnostics progress.

Apps inventory and risk visibility

Browse installed apps with search, source details, version data, install timestamps, size formatting, and risk signals for supported sideloaded or non-Play apps.

Diagnostics hub

The Tests area links into Device Check and Sensor Lab so users can run manual or automatic checks, see live sensor activity, and keep pass or fail status across app restarts.

Features

Documented to match the current Android app, not an old store listing.

Dashboard export to PDF

Selected dashboard sections can be exported into a shareable PDF report when users need a portable summary of device information.

Live Sensor Lab

Beyond static sensor metadata, the app includes a live monitoring surface with quick health status, recent history, and waveform-style feedback.

Device Check persistence

Diagnostics status survives app restarts, so users can come back later and still review which checks passed, failed, or remain unfinished.

Installed-app risk analysis

The Apps experience can highlight suspicious combinations for supported packages and also consume remotely managed blocklist signals.

Widgets and live summaries

RAM usage and device summary widgets keep key information visible from the home screen, with refresh support wired into the app lifecycle.

Remote-configured updates

Recommended and forced update rules can be delivered through Remote Config, with Play update handling or store fallback when a newer version is required.

In The App

The current product map is wider than just device specs.

Privacy

Clear permission and data-use language for Google Play review.

The privacy policy now covers Firebase Analytics, Crashlytics, Remote Config, local diagnostics state, PDF export behavior, installed-app visibility and risk analysis, Sensor Lab, update checks, and each documented manifest permission.

Most device information is read and displayed locally on the device itself.

Firebase services are used for analytics, crash reporting, config delivery, and update policy control.

Camera, microphone, vibration, package visibility, and activity recognition are tied to explicit user-facing diagnostics or sensor features.
